Documentation

nostr-zap provides embeddable Nostr zap UI components. Two main experiences: a zap viewer dialog and a legacy zap button flow. This page is a static HTML doc (so it works on your domain without needing Markdown rendering).

Embed (plain HTML)

1) Include the bundle

Add this near the end of your page. Since your site is served from zap.nostr.buzz, we use the full URL.

Copy script tag 
<script src="https://zap.nostr.buzz/dist/nostr-zap.js"></script>

2) Add a “View zaps” button

The library auto-wires any button[data-nzv-id] on click and opens a dialog.

Minimal viewer example 
<button
  data-nzv-id="npub1..."
  data-relay-urls="wss://relay.nostr.band,wss://relay.damus.io,wss://nos.lol"
  data-title=""
  data-zap-color-mode="true">
  View zaps
</button>

Viewer attributes

  • data-nzv-id (required): Nostr identifier (npub, nprofile, note, nevent, naddr).
  • data-relay-urls (required): comma-separated relay URLs.
  • data-title (optional): dialog title (empty string is allowed).
  • data-zap-color-mode (optional): "true" or "false".

JavaScript API (browser global)

After you include the bundle using a <script> tag, the library is available as window.nostrZap. In most cases you don’t need to call anything manually adding the data-* attributes to your buttons is enough.

  • window.nostrZap.initialize() — optional manual initialization
  • window.nostrZap.nostrZapView() — open the zap viewer dialog (alias)
  • window.nostrZap.nostrZap — legacy API object

Legacy zap buttons (still supported)

Older embeds remain compatible. The most common attributes are: data-npub, data-note-id, and data-relays.

Programmatic legacy API (browser global) 
// Low-level API object:
// window.nostrZap.nostrZap.init({ npub, noteId?, naddr?, relays?, buttonColor?, anon? })

// Convenience helpers:
await window.nostrZap.zapInit({
  npub: "npub1...",
  relays: "wss://relay.damus.io,wss://nos.lol",
});

window.nostrZap.zapInitTargets();
